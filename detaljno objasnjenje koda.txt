# `SelectionResult`

```python
@dataclass
class SelectionResult:
    selected_features: List[int]
    validation_score: float
    n_evaluated_subsets: int
    selection_time_sec: float
```

Ovo je mala klasa koja služi kao “paket rezultata”.

Kada SFS, SBS ili Random završe, vraćaju objekt koji kaže:

```text
selected_features       koji su featurei odabrani
validation_score        najbolji rezultat na validation skupu
n_evaluated_subsets     koliko je podskupova testirano
selection_time_sec      koliko je trajao odabir značajki
```

Primjer rezultata može biti:

```text
selected_features = [2, 5, 9]
validation_score = 0.91
n_evaluated_subsets = 120
selection_time_sec = 0.8
```

To se kasnije sprema u CSV.

# `_make_model(classifier)`

```python
def _make_model(classifier):
    return Pipeline([
        ("scaler", StandardScaler()),
        ("clf", clone(classifier)),
    ])
```

Ova funkcija napravi model koji ima dva koraka:

```text
1. StandardScaler
2. classifier
```

Dakle, prije treniranja klasifikatora podaci se skaliraju.

To je važno jer koristiš:

```text
KNN
SVM
```

Oba algoritma su osjetljiva na skalu značajki.

Primjer:

Ako jedan feature ima vrijednosti od 0 do 1, a drugi od 0 do 10000, model bi mogao dati preveliku važnost ovom drugom. Skaliranje to rješava.

`clone(classifier)` znači da se svaki put koristi nova, čista kopija klasifikatora. To je bitno jer wrapper puno puta trenira model na različitim subsetima.

---

# `_score_model(...)`

```python
def _score_model(y_true, y_pred, metric: str) -> float:
    if metric == "accuracy":
        return float(accuracy_score(y_true, y_pred))
    if metric == "macro_f1":
        return float(f1_score(y_true, y_pred, average="macro", zero_division=0))
    raise ValueError("metric mora biti 'accuracy' ili 'macro_f1'")
```

Ova funkcija računa metriku.

Može računati:

```text
accuracy
macro_f1
```

Ako je `metric == "accuracy"`, računa se accuracy.

Ako je `metric == "macro_f1"`, računa se macro-F1.

To je važno jer profesor traži da se feature selection radi jednom prema accuracy, a jednom prema F1.

Zato kasnije imaš:

```text
sfs_accuracy
sfs_macro_f1
sbs_accuracy
sbs_macro_f1
random_accuracy
random_macro_f1
```

# `evaluate_subset(...)`

```python
def evaluate_subset(
    classifier,
    X_train,
    y_train,
    X_val,
    y_val,
    subset,
    metric,
) -> float:
```

Ovo je jedna od najvažnijih funkcija u cijelom projektu.

Njezin posao je:

> uzmi jedan podskup featurea, treniraj model na train skupu i izmjeri rezultat na validation skupu.

Ovaj dio:

```python
model = _make_model(classifier)
```

napravi novi model:

```text
scaler + classifier
```

Ovaj dio:

```python
model.fit(X_train[:, subset], y_train)
```

trenira model samo na odabranim featureima.

Ako je:

```text
subset = [0, 3, 7]
```

onda se koristi samo:

```text
feature 0
feature 3
feature 7
```

Ovaj dio:

```python
y_pred = model.predict(X_val[:, subset])
```

radi predikcije na validation skupu, opet samo s tim featureima.

Na kraju:

```python
return _score_model(y_val, y_pred, metric)
```

vrati accuracy ili macro-F1.

Ovo je srce wrapper metode.

Wrapper metoda znači:

> kvaliteta feature subseta ocjenjuje se prema uspješnosti klasifikatora.

# `sfs_select(...)`

Ovo je implementacija SFS metode.

SFS znači:

```text
Sequential Forward Selection
```

Ideja:

> kreni bez featurea i dodaj featuree jedan po jedan.

---

## Početak SFS-a

```python
start = time.perf_counter()
n_features = X_train.shape[1]
remaining = list(range(n_features))
current_subset: List[int] = []
best_score = -np.inf
best_subset: List[int] = []
evaluated = 0
steps = 0
```

Ovdje se priprema algoritam.

`n_features` je broj featurea.

Ako dataset ima 22 featurea, onda je:

```text
n_features = 22
```

`remaining` je lista svih featurea koji još nisu odabrani.

Primjer:

```text
remaining = [0, 1, 2, 3, ..., 21]
```

`current_subset` je trenutni skup odabranih featurea. Na početku je prazan:

```text
current_subset = []
```

`best_score` je najbolji score dosad. Na početku je minus beskonačno jer još ništa nije testirano.

`evaluated` broji koliko je podskupova značajki testirano.

`steps` broji koliko je puta algoritam stvarno dodao novi feature.

---

## Glavna SFS petlja

```python
while remaining:
```

Petlja traje dok još postoje featurei koji se mogu dodati.

---

## Testiranje kandidata

```python
candidate_results = []

for feat in remaining:
    candidate_subset = current_subset + [feat]
    score = evaluate_subset(...)
    evaluated += 1
    candidate_results.append((score, candidate_subset, feat))
```

Ovdje se događa glavni SFS postupak.

Ako je trenutno:

```text
current_subset = [2]
remaining = [0, 1, 3, 4]
```

onda se testiraju kandidati:

```text
[2, 0]
[2, 1]
[2, 3]
[2, 4]
```

Za svaki kandidat se računa validation score.

Rezultati se spremaju u listu kao tuple:

```text
(score, candidate_subset, feat)
```

Primjer:

```text
(0.89, [2, 0], 0)
(0.91, [2, 1], 1)
(0.86, [2, 3], 3)
(0.88, [2, 4], 4)
```

---

## Odabir najboljeg kandidata

```python
step_best_score, step_best_subset, step_best_feat = max(candidate_results, key=lambda item: item[0])
```

Ova linija uzima kandidata s najvećim scoreom.

`lambda item: item[0]` znači:

> gledaj prvi element tuplea

A prvi element je score.

Dakle, ako imamo:

```text
(0.91, [2, 1], 1)
```

onda je:

```text
item[0] = 0.91
```

Zato `max(...)` bira kandidata s najvećim validation scoreom.

---

## Provjera poboljšanja

```python
if step_best_score > best_score + tol:
```

Ovdje algoritam pita:

> je li najbolji kandidat u ovom koraku bolji od dosad najboljeg rezultata?

Ako jest, feature se dodaje:

```python
current_subset = step_best_subset
remaining.remove(step_best_feat)
best_score = step_best_score
best_subset = current_subset.copy()
steps += 1
```

Dakle:

* novi trenutni subset postaje najbolji kandidat
* odabrani feature se miče iz `remaining`
* sprema se novi najbolji score
* sprema se novi najbolji subset

Ako nema poboljšanja:

```python
else:
    break
```

SFS staje.

To odgovara profesorovoj napomeni:

> ako nijedan feature više nije dodan, a prošli su svi, onda stajem

---

## Limit koraka

```python
if max_steps is not None and steps >= max_steps:
    break
```

Ovo je opcionalno.

Ako želiš ograničiti SFS da ne ide predugo, možeš zadati `--max-steps`.

Ako je `max_steps=None`, nema limita.

---

## Povrat rezultata

```python
return SelectionResult(...)
```

Na kraju SFS vraća:

* odabrane featuree
* najbolji validation score
* broj evaluiranih subsetova
* vrijeme trajanja

---

# `sbs_select(...)`

Ovo je implementacija SBS metode.

SBS znači:

```text
Sequential Backward Selection
```

Ideja:

> kreni od svih featurea i uklanjaj featuree jedan po jedan.

---

## Početak SBS-a

```python
start = time.perf_counter()
n_features = X_train.shape[1]
current_subset = list(range(n_features))
best_score = evaluate_subset(..., current_subset, metric)
best_subset = current_subset.copy()
evaluated = 1
steps = 0
```

Za razliku od SFS-a, SBS ne kreće od praznog skupa.

Kreće od svih featurea.

Ako dataset ima 22 featurea:

```text
current_subset = [0, 1, 2, ..., 21]
```

Odmah se računa score s punim skupom featurea:

```python
best_score = evaluate_subset(... current_subset ...)
```

Zato je `evaluated = 1`, jer je jedan subset već testiran.

---

## Glavna SBS petlja

```python
while len(current_subset) > 1:
```

Petlja traje dok ima više od jednog featurea.

Ne ide ispod 1 featurea jer model mora imati barem jedan ulaz.

---

## Testiranje uklanjanja svakog featurea

```python
candidate_results = []

for feat in current_subset:
    candidate_subset = [f for f in current_subset if f != feat]
    score = evaluate_subset(...)
    evaluated += 1
    candidate_results.append((score, candidate_subset, feat))
```

Ako je trenutni subset:

```text
[0, 1, 2, 3]
```

onda SBS testira:

```text
makni 0 -> [1, 2, 3]
makni 1 -> [0, 2, 3]
makni 2 -> [0, 1, 3]
makni 3 -> [0, 1, 2]
```

Za svaki od tih podskupova računa validation score.

---

## Odabir najboljeg uklanjanja

```python
step_best_score, step_best_subset, removed_feat = max(candidate_results, key=lambda item: item[0])
```

Bira ono uklanjanje koje daje najbolji validation score.

Ako je najbolji rezultat dobiven kada se makne feature 2, onda se feature 2 uklanja.

---

## Provjera poboljšanja

```python
if step_best_score > best_score + tol:
```

Ako uklanjanje poboljšava rezultat, prihvati ga:

```python
current_subset = step_best_subset
best_score = step_best_score
best_subset = current_subset.copy()
steps += 1
```

Ako nijedno uklanjanje ne poboljša rezultat:

```python
else:
    break
```

SBS staje.

Zato SBS često zadržava puno featurea. Ako uklanjanje ne pomaže, neće uklanjati samo da bi smanjio broj featurea.

---

# `random_select(...)`

Ovo je random wrapper metoda.

Ideja:

> slučajno generiraj više različitih podskupova featurea i vrati onaj koji daje najbolji validation score.

---

## Početak

```python
start = time.perf_counter()
rng = np.random.default_rng(random_state)
n_features = X_train.shape[1]
```

`rng` je generator slučajnih brojeva.

`random_state` omogućuje reproducibilnost. To znači da možeš ponoviti isti eksperiment i dobiti iste random subsetove.

---

## Raspon broja featurea

```python
if max_features is None:
    max_features = n_features

min_features = max(1, min(min_features, n_features))
max_features = max(min_features, min(max_features, n_features))
```

Ovo osigurava da random subset ima barem jedan feature i najviše sve featuree.

Ako dataset ima 22 featurea, random može odabrati:

```text
1 do 22 featurea
```

---

## Praćenje najboljeg subseta

```python
best_score = -np.inf
best_subset: List[int] = []
evaluated = 0
seen = set()
```

`seen` pamti već generirane subsetove, da se isti subset ne testira više puta.

---

## Glavna random petlja

```python
while evaluated < n_trials and attempts < max_unique_attempts:
```

Petlja traje dok nije evaluirano dovoljno random subsetova.

Ako je `n_trials = 484`, random pokušava evaluirati 484 različita subseta.

Kod `n2` pravila:

```text
n_trials = n_features * n_features
```

Primjeri:

```text
Parkinsons: 22 featurea -> 22*22 = 484 random subseta
Ionosphere: 34 featurea -> 34*34 = 1156 random subsetova
```

---

## Generiranje random subseta

```python
k = int(rng.integers(min_features, max_features + 1))
subset = sorted(rng.choice(n_features, size=k, replace=False).tolist())
```

Prvo se slučajno odabere broj featurea `k`.

Primjer:

```text
k = 5
```

Onda se slučajno odabere 5 različitih featurea.

Primjer:

```text
subset = [1, 4, 7, 13, 20]
```

`replace=False` znači da se isti feature ne može izabrati dvaput.

---

## Izbjegavanje duplikata

```python
subset_key = tuple(subset)

if subset_key in seen:
    continue

seen.add(subset_key)
```

Ako je isti subset već testiran, preskoči ga.

---

## Evaluacija subseta

```python
score = evaluate_subset(...)
evaluated += 1
```

Kao i kod SFS/SBS, model se trenira na train skupu i mjeri na validation skupu.

---

## Spremanje najboljeg random subseta

```python
if score > best_score:
    best_score = score
    best_subset = subset
```

Ako je novi subset bolji od svih ranijih, sprema se kao najbolji.

```text
SBS macro-F1
Random accuracy
Random macro-F1
```

---

## Random state

```python
random_state = base_seed + split_id
```

Svaki split ima drugačiji random seed, zato dobiješ 30 različitih podjela.

---

## Split podataka

```python
X_train, X_val, X_test, y_train, y_val, y_test = split_train_val_test(...)
```

Ovdje se radi 50/25/25 podjela.

---

## Train + validation za finalni model

```python
X_train_final = np.vstack([X_train, X_val])
y_train_final = np.concatenate([y_train, y_val])
```

Nakon što wrapper odabere featuree, finalni model se trenira na train + validation.

To je ispravno jer:

* validation je korišten za izbor featurea
* ali finalni model može iskoristiti sve podatke osim testa

Test ostaje netaknut do kraja.

# Zašto je ovo metodološki ispravno?

Zato što test skup nije korišten za izbor featurea.

Da si featuree birao na test skupu, rezultat bi bio pristran.

Kod tebe:

```text
train      → treniranje kandidata
validation → izbor featurea
test       → konačna evaluacija
```

To je čisto i obranjivo.

# Što je u kodu posebno važno za profesora?

## 1. Stratify

Čuva omjer klasa.

Važno kod Parkinsonsa jer je neuravnotežen:

```text
klasa 0: 48
klasa 1: 147
```

---

## 2. 30 splitova

Ne oslanjaš se na jednu slučajnu podjelu.

Zato računaš:

```text
mean
std
min
max
```

---

## 3. Dvije metrike

Koristiš:

```text
accuracy
macro-F1
```

Accuracy može biti varljiva kod neuravnoteženih podataka.

Macro-F1 bolje pokazuje ponašanje po klasama.

---

## 4. Random wrapper

Random metoda generira slučajne subsetove i bira najbolji po validation scoreu.

To je wrapper jer se subseti ocjenjuju pomoću modela.

---

## 5. Scaling

KNN i SVM trebaju skaliranje.

Zato svaki model ima:

```text
StandardScaler + classifier
```
