# 1. RANDOM WRAPPER — što profesor zapravo misli?

Profesor je rekao:

> “prvo krenit s random, broj od 10 do n kolko cu feature odabrat feature biram nasumicno, 1 do n koji feature, nxn puta”

To zapravo znači:

---

## Ideja random wrapper metode

Ako dataset ima:

```text
n = broj featurea
```

npr.

```text
22 featurea
```

onda:

1. nasumično odabereš koliko featurea će subset imati

npr.

```text
k = 5
```

2. onda slučajno odabereš kojih 5 featurea

npr.

```text
[1, 4, 7, 13, 20]
```

3. treniraš model samo na tim featureima
4. izmjeriš validation accuracy ili F1
5. ponavljaš to puno puta
6. zadržiš najbolji subset

---

## Što znači nxn?

Profesor je predložio:

```text
n² random subsetova
```

Ako dataset ima:

```text
22 featurea
```

onda:

```text
22 * 22 = 484 random subseta
```

Ako ima:

```text
34 featurea
```

onda:

```text
34 * 34 = 1156 subsetova
```

Točno to si implementirao.

---

# 2. Kako si to riješio u kodu

Profesoru možeš reći:

> Kod random wrapper metode za svaki pokušaj slučajno generiram podskup značajki. Prvo nasumično biram broj značajki od 1 do n, zatim slučajno biram konkretne featuree bez ponavljanja. Za svaki subset treniram klasifikator na train skupu i evaluiram na validation skupu. Proces ponavljam n² puta i zadržavam subset koji daje najbolji validation rezultat.

---

## Snippet koji možeš pokazati

```python
k = rng.integers(1, n_features + 1)

subset = rng.choice(
    n_features,
    size=k,
    replace=False
)
```

Objasni:

```text
k = koliko featurea uzimam
subset = koji featurei su odabrani
```

---

# 3. STRATIFY — najvažniji koncept

Ovo profesori često pitaju.

---

## Što je problem bez stratify?

Ako dataset ima:

```text
klasa 0 = 10 uzoraka
klasa 1 = 100 uzoraka
```

i napraviš random split bez stratify:

može se dogoditi:

```text
train:
  2 uzorka klase 0
  75 uzoraka klase 1

test:
  8 uzoraka klase 0
  25 uzoraka klase 1
```

što je loše.

---

## Što radi stratify?

`stratify=y` čuva omjer klasa u svim skupovima.

Ako original ima:

```text
10% klase 0
90% klase 1
```

onda train, validation i test također imaju približno isti omjer.

---

# Snippet za stratify

Ovo OBAVEZNO pokaži profesoru.

```python
X_train_val, X_test, y_train_val, y_test = train_test_split(
    X,
    y,
    test_size=0.25,
    stratify=y,
    random_state=random_state
)
```

Objasni:

> Stratify koristim kako bi se pri svakom splitu zadržao približno isti omjer klasa kao u originalnom datasetu.

# 4. TRAIN / VALIDATION / TEST — najvažnija stvar

Profesor ti je zapravo opisao standardni ML eksperimentalni protokol.

---

# Kako si ti to riješio

## Korak 1

Prvo odvajaš test skup.

```python
X_train_val, X_test, y_train_val, y_test = train_test_split(
    X,
    y,
    test_size=0.25,
    stratify=y,
    random_state=random_state
)
```

Dobiješ:

```text
75% train+validation
25% test
```

---

## Korak 2

Zatim train+validation dijeliš opet:

```python
X_train, X_val, y_train, y_val = train_test_split(
    X_train_val,
    y_train_val,
    test_size=1/3,
    stratify=y_train_val,
    random_state=random_state + 10000
)
```

Dobiješ:

```text
50% train
25% validation
25% test
```

---

# Kako to objasniti profesoru

> Test skup odvajam odmah na početku i ne koristim ga tijekom feature selection procesa. Wrapper metode koriste train skup za treniranje kandidata i validation skup za odabir najboljeg subseta. Tek nakon odabira featurea finalni model treniram na train+validation podacima i evaluiram na test skupu.

To zvuči jako dobro i metodološki ispravno.

# 7. SFS — što radi

Profesor će možda pitati:

> “Objasni mi SFS.”

---

## Kratko objašnjenje

> Sequential Forward Selection kreće od praznog skupa značajki i u svakom koraku dodaje onu značajku koja najviše poboljšava validation rezultat klasifikatora. Proces staje kada nijedna preostala značajka više ne poboljšava metriku.

---

# Snippet

```python
candidate_subset = current_subset + [feat]
```

Objasni:

> U svakom koraku isprobavam dodavanje svakog preostalog featurea.

---

# 8. SBS — što radi

---

## Kratko objašnjenje

> Sequential Backward Selection kreće od svih značajki i u svakom koraku uklanja onu čije uklanjanje daje najbolji validation rezultat. Proces staje kada uklanjanje više ne poboljšava metriku.

---

# Snippet

```python
candidate_subset = [
    f for f in current_subset
    if f != feat
]
```

---
